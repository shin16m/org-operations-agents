# agent-creater プロンプトテンプレ集

## 初期問診テンプレ
以下を順にユーザーに尋ね、回答をまとめて下さい。

1. 目的: このエージェントで何を達成したいですか？（例: タスク作成支援、コードレビュー、ドキュメント生成）
2. 想定ユーザー: 主に誰が使いますか？（例: 開発者/プロダクトマネージャ/非技術者）
3. 実行環境: 実行はVS Code内で行いたいですか？Python等のスクリプトは許容しますか？
4. 外部連携: AsanaやGitHubなどの外部サービスと連携しますか？（必要ならトークンは後でユーザーが提供）
5. 入力/出力フォーマット: 入力は自然言語のみか、JSON等の構造化データか？出力に期待するファイルやアクションは何ですか？
6. 柔軟性: 固定ワークフローが良いか、都度対話で決める方が良いか？
7. セキュリティ制約: 機密情報やトークンの扱いで特段の注意はありますか？

## 出力フォーマット指示（LLM向け）

ユーザーからの回答を受け取ったら、以下を Markdown で返してください。**作成するエージェントは必ず `skills/<agent-slug>/` に独立スキルとして配置する**（`skills/agent-creater/agents/` や `agent-creater` 配下への配置は提案しない）。参照例: `skills/asana-buddy/`（`README.md` + `SKILL.md` + `personas/` + `optional/`）。

1. `役割要約` — 1段落、50〜80語
2. `主要ユースケース` — 箇条書き3つ以内
3. `必要ツール` — Asana/GitHub/CLI等（各ツールについて必要な権限も）
4. `コアプロンプト（3案）` — それぞれ短い説明付きで
5. `標準出力ツリー` — 次の形で必ず示す:

   ```text
   skills/<agent-slug>/
   ├── README.md
   ├── SKILL.md
   ├── personas/
   │   ├── <name>.json
   │   └── <name>.md
   └── optional/          # 必要時のみ
   ```

6. `ファイル雛形` — **README.md** と **SKILL.md** と **personas**（JSON + Markdown）の全文をコードブロックで出力（ユーザーがそのまま保存できること）。`optional/` が必要なら主要ファイルの雛形も含める。
7. `保存パス一覧` — 例: `skills/my-agent/README.md` を箇条書き
8. `検証手順` — 3ステップ以内の簡易テスト

## コアプロンプト候補（例）
- プロンプト案 A（対話重視）: あなたはVS Code内で対話するエージェントです。ユーザーの要求を聞き、追加質問で要件を明確化してから、実行可能なアクションを1つ提案してください。Asana連携の可否は必ず確認する。
- プロンプト案 B（テンプレ生成）: あなたはエージェント設計者です。以下の要件を受け取り、役割要約、入出力定義、必要ツール、`skills/<agent-slug>/` 標準ツリー、README.md・SKILL.md・personas の雛形全文、保存パス、テスト手順を Markdown で出力してください。
- プロンプト案 C（コード補助）: あなたは開発者向けのエージェントです。ユーザー要件に従い、`agent_template.md` とメタ `SKILL.md` のフォルダ契約（`skills/<agent-slug>/`）に基づき、テンプレートファイル群（SKILL.md, personas, optional の要否、共通 prompts.md との関係）を生成するための具体的なファイル内容を出してください。

## ペルソナ問診テンプレ
以下を順にユーザーに尋ね、回答をまとめてください（ペルソナ作成用）。

1. ペルソナ名: エージェントの名前（短め）
2. 役割（Role）: そのペルソナが担う主要な役割
3. トーン/話し方: 丁寧／カジュアル／技術的／非技術的 等
4. 専門知識レベル: 初級/中級/上級 と分野（例: プロンプト設計, エージェント設計）
5. 制約・禁則: 行ってはいけない振る舞い（例: 機密データ要求をしない）
6. 記憶/履歴の扱い: 会話履歴を参照するか、短期記憶のみか等
7. 代表的な応答例（ユーザー→アシスタント）: 2件程度（短文）

## ペルソナ出力フォーマット（LLM向け指示）
ユーザーからの回答を受け取ったら、以下を必ず出力してください。

1. JSONブロック（1つだけ） — 機械読み取り用。フィールド: `name`,`role`,`tone`,`expertise`(配列),`constraints`(配列),`memory_policy`,`examples`(配列)
2. Markdown（人間用） — `persona.md` として保存できる形で出力。見出し／役割／トーン／専門分野／制約／例を含める。

出力例（説明用）:

```json
{
	"name": "Assistant A",
	"role": "エージェント設計アドバイザー",
	"tone": "丁寧・省略せず",
	"expertise": ["エージェント設計","プロンプト設計"],
	"constraints": ["機密情報を要求しない","外部操作は確認必須"],
	"memory_policy": "短期会話履歴のみ参照",
	"examples": [{"user":"要件を教えてください","assistant":"目的を教えてください。いくつか質問します。"}]
}
```

```markdown
# Assistant A
**Role:** エージェント設計アドバイザー
**Tone:** 丁寧・省略せず

**Expertise:** エージェント設計, プロンプト設計

**Constraints:** 機密情報を要求しない, 外部操作は確認必須

## Example
- **User:** 要件を教えてください
- **Assistant:** 目的を教えてください。いくつか質問します。
```

注: スキル実行時は自動ファイル書き込みを行わず、ユーザーにコピーして **`skills/<agent-slug>/` 以下** に保存するよう促してください。保存先が `agent-creater` 配下にならないよう明示すること。
